# SMART CLASS AXIS - COMPLETE API ENDPOINTS LIST
# Based on Frontend Requirements Analysis

## 🔐 AUTHENTICATION & AUTHORIZATION APIs

### User Authentication
POST /api/auth/login
- Body: { email, password, role }
- Response: { token, user, role, permissions }

POST /api/auth/logout
- Headers: { Authorization: Bearer <token> }
- Response: { message: "Logged out successfully" }

POST /api/auth/refresh
- Headers: { Authorization: Bearer <token> }
- Response: { token, expiresIn }

### User Registration (HOD Only)
POST /api/auth/signup
- Body: { fullName, email, password, collegeCode, role: "hod" }
- Response: { user, message: "Account created successfully" }

### Profile Management
GET /api/auth/profile
- Headers: { Authorization: Bearer <token> }
- Response: { user profile data }

PUT /api/auth/profile
- Headers: { Authorization: Bearer <token> }
- Body: { fullName, email, phone, address }
- Response: { updated user profile }

## 🔒 SECURITY & PERMISSIONS

### Role-Based Access Control
- HOD: Full access to all APIs
- Professor: Access to student management, materials, assignments, results, communication
- Student: Read-only access to most data, can submit assignments and ask questions

### API Rate Limiting
- Authentication APIs: 5 requests per minute
- CRUD Operations: 100 requests per minute
- File Upload: 10 requests per minute
- Search APIs: 50 requests per minute

### Data Validation
- All POST/PUT requests require input validation
- File uploads limited to specific formats and sizes
- Email and phone number format validation
- Date range validation for reports

## 🗂️ FILE STRUCTURE RECOMMENDATIONS

### API Versioning
- Base URL: /api/v1/
- All endpoints prefixed with version

### Endpoint Naming Convention
- Use RESTful conventions
- Use plural nouns for collections
- Use HTTP methods appropriately (GET, POST, PUT, DELETE)
- Use query parameters for filtering and pagination
